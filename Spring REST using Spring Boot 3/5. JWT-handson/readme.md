# JWT Authorization Filter

## Overview

This hands-on implements authorization of REST APIs using JSON Web Tokens (JWT).

Instead of validating Basic Authentication credentials on every request, a JWT generated by the authentication service is validated using a custom Spring Security filter.

## How It Works

1. The client first authenticates using Basic Authentication.

```
GET /authenticate
```

2. The server generates a JWT.

3. The client includes the JWT in every subsequent request.

```
Authorization: Bearer <jwt-token>
```

4. JwtAuthorizationFilter intercepts every request.

5. The filter:

- Reads the Authorization header.
- Checks whether it starts with `Bearer`.
- Validates the JWT using the secret key.
- Extracts the username.
- Marks the request as authenticated.

## Security Flow

```
Client
   │
   │  Basic Authentication
   ▼
Authentication Service
   │
   │
   ▼
Generate JWT
   │
   │
Bearer Token
   │
   ▼
JwtAuthorizationFilter
   │
   ▼
Spring Security
   │
   ▼
REST Controller
```

## Testing

Generate a token:

```bash
curl -u user:pwd http://localhost:8090/authenticate
```

Response

```json
{
  "token":"<generated-jwt>"
}
```

Use the generated token:

```bash
curl -H "Authorization: Bearer <generated-jwt>" \
http://localhost:8090/countries
```
